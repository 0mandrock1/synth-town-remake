# Development Workflow

## Цель
Держать разработку предсказуемой: один этап за раз, жёсткие контракты, проверка тестами.

## Шаги
1. Определи текущий этап в `stages/STAGE_*.md`.
2. Проверь зависимые API в `docs/claude/CONTRACTS.md`.
3. Реализуй MUST-пункты.
4. Запусти тесты этапа из `tests/`.
5. Реализуй SHOULD-пункты (если есть время).
6. Повтори тесты и только после этого переходи дальше.

## Правило изменений
- Сигнатуры API прошлых этапов не ломать.
- Изменения вне этапа допускаются только как backward-compatible расширение.
